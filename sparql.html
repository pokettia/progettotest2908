Query utilizzate

Le query utilizzate sono state fondamentali per interrogare i dati disponibili e ottenere informazioni specifiche sui monumenti analizzati. Qui riportiamo tre esempi principali, con la spiegazione del loro scopo e funzionamento.

Query SELECT DISTINCT

Questa query viene utilizzata per estrarre un insieme di risultati unici (senza duplicati) da un dataset RDF. Nel nostro caso serve per cercare i monumenti di tipo ArchitecturalOrLandscapeHeritage che hanno determinate etichette (label), denominazioni o titoli, filtrando solo quelli che contengono il nome di una porta specifica e la citt√† di Bologna.

Esempio preso da porta_porta_maggiore.html ::

PREFIX rdf:  <http://www.w3.org/1999/02/22-rdf-syntax-ns#>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>
PREFIX arco: <https://w3id.org/arco/ontology/arco/>
PREFIX arco-core: <https://w3id.org/arco/ontology/core/>
PREFIX cis:  <http://dati.beniculturali.it/cis/>

SELECT DISTINCT ?porta ?label
WHERE {
  ?porta rdf:type arco:ArchitecturalOrLandscapeHeritage .
  { ?porta rdfs:label ?label }
  UNION { ?porta cis:name ?label }
  UNION { ?porta arco-core:hasTitle ?title .
           ?title rdfs:label ?label }
  UNION { ?porta arco-core:hasDenomination ?den .
           ?den rdfs:label ?label }
  FILTER (REGEX(STR(?label), "Porta Maggiore", "i"))
  FILTER (REGEX(STR(?label), "Bologna", "i"))
}
LIMIT 100

Query DESCRIBE

La query DESCRIBE serve per ottenere tutte le informazioni RDF collegate a una specifica risorsa. In questo caso, a partire da un identificativo unico, vengono restituite tutte le propriet√† e i valori relativi al monumento in questione, permettendo di avere una descrizione dettagliata dell'entit√†.

Esempio preso da porta_porta_san_donato.html :

PREFIX arco: <https://w3id.org/arco/ontology/arco/>

DESCRIBE ?s
WHERE {
  ?s arco:uniqueIdentifier "0800241601" .
}
ORDER BY ?s
LIMIT 200
