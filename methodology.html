Website Requirements (for your project)

  • Publish the entire site on GitHub Pages.
  • Make all code and materials public in a GitHub repository.
  • Include a homepage (index.html) with a project title, abstract, and team members.
  • Add a simple, consistent menu visible in the same position on every page.
  • Sections to include:
    • Topic: context and background.
    • Methodology: how you identified the gap; overall process and tools.
    • Methodological Steps: pages with SPARQL queries and prompts, example outputs, challenges and solutions, and differences observed across LLMs.
    • Report: methodology to build your stories, tools (e.g., Melody), data sources.
    • Challenges: what you faced and how you addressed them.
  • Whenever you mention a web-browsable topic, include an outbound link.

Exam & Delivery

  • Present the project with slides covering methodology and outcomes.
  • Follow the report‚Äôs outline, but use examples to illustrate steps (not everything must be in the slides).
  • Timing: max 2 minutes per team member.
  • Upload slides to GitHub 48 hours before the exam and include a link to the website on the title slide.
